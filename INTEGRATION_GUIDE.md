# IMDb Real Data Integration — Step-by-step Guide

## Project Structure

```
C:.
├── api.py                         ← new — Flask server
├── scripts/
│   └── build_db.py                ← new — builds the database
├── data/
│   ├── raw/
│   │   └── imdb_english.csv       ← your source file
│   └── processed/
│       └── imdb.db                ← generated by build_db.py
└── js/
    └── data/
        └── all_decades.js         ← replaced with the live version
```

---

## Step 1 — Install Python dependencies

```bash
pip install flask flask-cors
```

---

## Step 2 — Build the SQLite database

```bash
python scripts/build_db.py \
  --csv data/raw/imdb_english.csv \
  --db  data/processed/imdb.db
```

The script will:
- Read every row of the CSV
- Compute the `decade` column from each year (e.g. 1994 → 1990)
- Create indexes on `(decade)` and `(decade, numVotes DESC)` for instant queries
- Print a per-decade summary at the end

## Available endpoints

| Method | URL | Description |
|--------|-----|-------------|
| `GET` | `/api/decades` | List all decades with basic stats |
| `GET` | `/api/decade/1990s` | Full decade data (top 20 films) |
| `GET` | `/api/decade/1990s/film/tt0111161` | Single film detail |

---

## Response format — `/api/decade/1990s`

```json
{
  "decade":     "1990s",
  "avgRating":  7.85,
  "avgVotes":   412000,
  "avgRuntime": 117,
  "minRating":  7.2,
  "totalFilms": 284,
  "topGenres": [
    { "genre": "Drama",  "decadePct": 75 },
    { "genre": "Crime",  "decadePct": 40 }
  ],
  "films": [
    {
      "rank":     1,
      "tconst":   "tt0111161",
      "title":    "The Shawshank Redemption",
      "year":     1994,
      "rating":   9.3,
      "votes":    2800000,
      "runtime":  142,
      "genres":   ["Drama"],
      "director": null,
      "actor":    null,
      "summary":  null,
      "poster":   null
    }
  ]
}
```

> The `director` fields depend on what your CSV contains. If absent, the frontend displays the field as empty.

## Notes on CORS and deployment

- During development, HTML pages are opened directly as files (`file://`), so Flask must respond with CORS headers. `flask-cors` handles this automatically.
- In production, serve both the pages and the API from the same domain (e.g. with nginx) and you can remove the `flask-cors` dependency.
